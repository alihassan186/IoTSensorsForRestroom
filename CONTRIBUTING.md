# Contributing Guide

Thanks for contributing.

## Workflow

1. Fork the repository
2. Create a feature branch from `main`
3. Keep changes focused and small
4. Test locally
5. Open a pull request

## Branch Naming

Use one of:

- `feat/<short-description>`
- `fix/<short-description>`
- `docs/<short-description>`
- `chore/<short-description>`

## Commit Message Style

Recommended (Conventional Commits):

- `feat: add rule notification dedup`
- `fix: correct occupancy threshold parsing`
- `docs: update setup instructions`

## Pull Request Checklist

- [ ] Code builds/runs locally
- [ ] No unrelated changes included
- [ ] Docs updated where relevant
- [ ] Secrets are not introduced
- [ ] Clear PR title and description

## Code Style

- Follow existing project style
- Prefer clear names over short abbreviations
- Avoid unnecessary refactors in unrelated areas

## Reporting Bugs

Please include:

- Steps to reproduce
- Expected behavior
- Actual behavior
- Relevant logs/errors
- Environment (Python version, OS)
